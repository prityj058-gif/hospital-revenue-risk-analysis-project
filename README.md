# 🏥 Hospital Revenue & Risk Analysis Dashboard

## 📌 Project Overview
This project analyzes hospital patient data to identify revenue trends, patient risk levels, and operational insights.

The project was completed using **Excel for data cleaning and analysis** and **Power BI for interactive dashboard visualization**.

---

## 📊 Objectives
- Analyze hospital revenue patterns
- Identify high-risk patients
- Understand patient demographics
- Track admission types and medical conditions
- Monitor hospital performance metrics

---

## 🛠 Tools Used
- Microsoft Excel (Data Cleaning & Analysis)
- Power BI (Dashboard Visualization)
- GitHub (Project Documentation)

---

## 📂 Dataset Features

The dataset includes the following information:

- Patient Name
- Age
- Gender
- Blood Type
- Medical Condition
- Admission Date
- Discharge Date
- Doctor
- Hospital
- Room Number
- Insurance Provider
- Billing Amount
- Medication
- Test Results

---

## 🔧 Data Cleaning Steps

The following preprocessing steps were performed in Excel:

- Removed duplicate records
- Fixed date formats
- Checked missing values
- Created calculated columns:
  - Length of Stay
  - Age Group
  - Revenue Per Day
  - Risk Score
  - Risk Category

---

## 📈 Dashboard Insights

The Power BI dashboard provides the following insights:

- Total Patients Count
- Total Hospital Revenue
- Average Patient Stay (Days)
- High Risk Patients Count
- Revenue by Medical Condition
- Patient Distribution by Age Group
- Patient Distribution by Gender
- Test Result Distribution
- Revenue Trend by Year & Month

---
 

## 💡 Key Insights

- Senior patients represent a large portion of hospital admissions.
- Certain medical conditions generate higher revenue.
- Emergency admissions often have longer hospital stays.
- High-risk patients require additional monitoring.
